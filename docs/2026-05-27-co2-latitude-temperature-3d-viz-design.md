# 全球气候演变三维探索：CO₂、纬度与温度异常的交叉分析 (1880–2025)

设计文档 · 2026-05-27

## 1. 动机与问题

气候变化是全人类面临的最严峻挑战之一。"全球平均气温上升"和"二氧化碳浓度上升"两条曲线读者已经看过无数次，但有几个关键结论很难用二维静态图表达：

1. 温度上升不是均匀的——**极地放大效应**让北极升温速度是低纬度的 3–4 倍；
2. CO₂ 浓度与温度的关系**对不同纬度的强度不同**；
3. **时间维度**上的演化（哪一年是转折点、哪些年是异常年）需要动态对比。

本可视化通过把 CO₂、纬度、温度异常三个变量映射到 3D 空间，并用联动多视图 + 时间动画支持探索式分析，让用户能自己发现并验证这些结论。

**核心分析问题**：
- Q1：不同纬度带的升温速度差异有多大？极地放大效应是否随时间增强？
- Q2：CO₂ 浓度上升与全球各纬度的温度变化有怎样的相关结构？
- Q3：哪些年份是温度异常年（如 1998、2016、2023、2024）？这些异常在哪些纬度最显著？

## 2. 目标用户

- **高校气候科学/环境科学/可视化课程学生**——需要把抽象的"全球变暖"概念落地到可探索的数据
- **对气候变化有兴趣的公众**——希望看到比新闻标题更有说服力的可视证据
- **科普传播者/教师**——需要直观的演示工具向他人解释极地放大、CO₂-温度耦合等现象

**用户特性假设**：会读基础英文术语，但偏好中文界面；具备基本图表阅读能力；预期能在桌面浏览器（≥1280×800 分辨率）上访问。

## 3. 数据

### 3.1 数据来源

| 文件 | 来源 | 时段 | 单位 | 备注 |
|---|---|---|---|---|
| `co2_annmean_gl.csv` | NOAA GML | 1979–2025 | ppm | 年均全球 CO₂ + 不确定度 |
| `ZonAnn.Ts+dSST.csv` | NASA GISTEMP v4 | 1880–2025 | °C | 相对 1951–1980 基准的纬度带温度异常 |

### 3.2 Munzner 数据抽象

- **Dataset type**：Tables（两张数据表，通过 year 关联）
- **Attribute types**：
  - `year`：序数（ordered, sequential）
  - `co2_mean`、`co2_unc`：定量（quantitative, sequential, ratio）
  - `temp_anomaly[lat_band]`：定量（quantitative, diverging，因为有正负偏离）
  - `lat_band`：序数（ordered, sequential——从南极到北极有自然顺序）

### 3.3 数据处理

1. **CO₂ 表**：跳过表头注释行，读为 `(year, mean, unc)`
2. **温度表**：读全部列，保留 8 个细分纬度带 + Glob/NHem/SHem 作为可选过滤项
3. **关联**：以 `year` 为主键，左联接（保留 1880–2025 完整温度时段，CO₂ 在 1979 前为 NaN）
4. **缺失值处理**：温度表中可能出现 `***` 占位符（GISTEMP 表示数据缺失），转为 NaN；CO₂ 1979 前显式留空
5. **生成 long 表用于 3D 动画**：将 8 个纬度列 melt 成 `(year, lat_band, temp_anomaly)`，再左联 CO₂

## 5. 视觉编码（Munzner）

### 5.1 总体 Idiom

**Linked multi-view dashboard with animated 3D scatterplot** —— 主视图为 3D 散点动画，辅以 2D 热图与 2D 时序图，时间滑块联动所有面板。

### 5.2 主视图：3D 散点 + 时间动画

| 通道 (Channel) | 编码变量 | 类型 | 说明 |
|---|---|---|---|
| **X 轴空间位置** | CO₂ 浓度 (ppm) | 定量 | 范围 ~336–426 ppm |
| **Y 轴空间位置** | 纬度中点 (-77→77) | 序数 | 8 个离散刻度对应 8 个带 |
| **Z 轴空间位置** | 温度异常 (°C) | 定量发散 | 范围约 -1.5–+3.0 |
| **颜色 (Hue)** | 温度异常 (°C) | 定量发散 | RdBu_r 发散调色板，冗余强调 Z 值 |
| **Mark** | 球点 (point) + 历史轨迹 | — | 当前年 8 个高亮球点；过去 5 年用半透明小点拖尾，增强密度感 |
| **Animation frame** | 年份 (1979→2025) | 序数 | 47 帧 |

### 5.3 辅视图 A：温度热图

| 通道 | 编码 | 说明 |
|---|---|---|
| X | 年份 1880–2025 | 完整温度史 |
| Y | 纬度带（8 行，南极→北极） | 与 3D Y 轴对齐 |
| Color | 温度异常 | 同主视图的 RdBu_r 调色板，便于对照 |
| 黄色竖线 | 当前年份指示 | 与主滑块联动 |

### 5.4 辅视图 B：CO₂ 时序图

| 通道 | 编码 | 说明 |
|---|---|---|
| X | 年份 1880–2025 | 完整范围；1979 前为虚线"无数据" |
| Y | CO₂ 浓度 ppm | 1979–2025 实线 |
| 阴影带 | ± 不确定度 (unc 列) | 轻浅色填充 |
| 黄色游标 | 当前年份点 | 与主滑块联动 |

### 5.5 配色

- **温度**：`RdBu_r` 发散，中点 0°C 为白色，负为蓝，正为红。对色盲（红绿色盲）友好——选 RdBu 而非 RdYlGn。
- **CO₂**：单色亮蓝 `#66c2ff`。
- **当前年份游标**：暖黄 `#fbbf24`，与温度色阶可区分。
- **主题**：`plotly_dark` 暗背景——强调发光的发散色，符合科学可视化惯例。

## 6. 交互

| 交互 | 触发 | 效果 |
|---|---|---|
| **拖动时间滑块** | 鼠标 | 3D 点动画到目标年；热图竖线移动；CO₂ 游标移动 |
| **播放按钮 ▶** | 点击 | 自动 1979→2025 滚动，约 200ms/帧；再次点击暂停 |
| **速度选择 ▼** | 下拉 | 0.5× / 1× / 2× / 4× |
| **悬停 3D 点** | 鼠标 | 浮动提示：年份 / 纬度带 / 温度异常 / 同年 CO₂ |
| **点击图例（纬度）** | 单击 | 隐藏/显示该纬度的所有点 |
| **双击图例** | 双击 | 隔离单一纬度 |
| **拖拽 3D 主视图** | 鼠标 | 旋转视角；双击主视图空白处恢复初始视角 |
| **滚轮 / 触控板** | 滚动 | 缩放 3D |
| **下拉切换粒度** | 控件 | 8 细带 ↔ 3 粗带 ↔ 半球 (NHem/SHem/Glob) |
| **重置按钮** | 点击 | 视角、滑块、过滤全部回到初始状态 |

**联动方向**：滑块为单一真相源 → 3D + 热图 + CO₂ 同步更新。3D 旋转、悬停、图例操作不反向影响其他面板（避免循环复杂度）。

## 7. 设计理由（引用 Munzner、感知与课程文献）

- **3D 位置 > 颜色长度**：Mackinlay/Cleveland 的可视通道有效性研究指出对定量变量，空间位置精度最高。我们把三个核心定量变量都映射到 3D 位置以最大化可读性。
- **冗余编码温度（颜色 + Z）**：Munzner 建议在通道带宽允许时冗余编码可增强读图效率，特别是在 3D 视角发生遮挡时颜色能补偿丢失的 Z 信息。
- **发散调色板而非彩虹**：Borland & Taylor (2007) 等研究指出彩虹色阶不符合感知线性度，发散色阶（RdBu）对"偏离基准"型数据更合适。
- **动画 + 静态热图并存**：动画对发现演化型模式有效但不利于精确比较；静态热图提供"宏观地图"，二者互补——Heer et al. (2010) 的多视图原则。
- **暗背景**：减少眼疲劳并让发散色更亮眼；与天文/地球观测可视化领域惯例一致。

## 8. 技术实现

### 8.1 技术栈

- **运行环境**：Python 3.12（已在 `.python-version`）
- **依赖管理**：`uv`（已是 PEP 621 `pyproject.toml`）
- **核心库**：
  - `plotly` ≥ 5.18 —— 3D scatter + frames + subplots
  - `pandas` ≥ 2.2 —— 数据 IO 与变形
  - `numpy` ≥ 1.26 —— 数值
- **构建脚本**：单一 `build.py` 读 CSV → 生成 `index.html`

### 8.2 代码结构

```
Finel-Assignment/
├── data/                    # 移动 CSV 到此（保留原 CSV 文件）
│   ├── co2_annmean_gl.csv
│   └── ZonAnn.Ts+dSST.csv
├── src/
│   ├── __init__.py
```

每个模块单一职责，便于独立测试与替换。

### 8.3 关键 Plotly 模式

- **子图布局**：`make_subplots(rows=2, cols=2, specs=[[{'type':'scene'},{'type':'xy'}],[{'type':'xy','colspan':2},None]])` —— Plotly 中热图属于 `'xy'` 类型
- **3D 散点动画**：`go.Scatter3d(...) + go.Figure(frames=[...])`；每帧 8 个点
- **滑块联动热图竖线 + CO₂ 游标**：通过每帧同时更新三个 trace 实现
- **导出**：`fig.write_html('index.html', include_plotlyjs='cdn', full_html=True)`

### 8.4 部署

1. 创建 GitHub repo（公开），推送代码 + 生成的 `index.html`
2. Settings → Pages → Source = `main` branch / root
3. URL 形如 `https://<用户名>.github.io/<repo>/`
4. 在报告中引用该 URL

### 8.5 性能预估

- 数据规模：8 纬度 × 47 年 ≈ 376 个 3D 点；热图 8 × 146 ≈ 1168 格子
- 输出文件大小：约 4–8 MB（含 CDN plotly.js 引用）
- 浏览器首次渲染 < 2 秒（中等笔记本）

## 9. 范围外（YAGNI）

明确不做：

- 不做 3D 曲面或飘带版本（已在选型阶段排除，作业不需要多版本）
- 不拼接冰芯 CO₂——保持单一数据源
- 不做月度数据——年度足以表达长期趋势
- 不做反向交互（3D 旋转影响其他面板）——避免循环复杂度
- 不做用户登录、保存视图、分享链接——课程作业范围外
- 不做移动端响应式——目标用户预期在桌面浏览

## 10. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 3D 视角下遮挡导致点被挡住 | 用户读图困难 | 颜色冗余编码温度；提供"重置视角"；图例可隔离 |
| 散点稀疏（每帧仅 8 点）显得信息量小 | 用户感觉不充实 | 已纳入设计：当前年高亮 + 前 5 年半透明拖尾（见 5.2） |
| GH Pages 大文件加载慢 | 用户体验差 | `include_plotlyjs='cdn'` 走 CDN；index.html 控制在 < 10MB |
| 课程评审无法运行代码 | 失分 | 直接提供已构建的 `index.html` 在仓库根；附带 README 截图 |
